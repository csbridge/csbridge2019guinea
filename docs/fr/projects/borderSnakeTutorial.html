Serpent frontière

Écrit par Lisa Yan

Référence: Référence graphique
ArrayLists

Ceci est un programme de bonus! C'est un peu difficile, mais terminer ce programme vous aidera certainement à écrire votre projet final.

Ecrivez un programme graphique qui déplace un serpent dans le sens antihoraire autour du bord de la toile.

BoîteFrontière

Cet exercice suppose que vous avez terminé BorderBox. La logique est similaire (il n’ya qu’une astuce supplémentaire), il est donc très utile de connaître la configuration générale du problème. Si vous n'avez pas encore terminé BorderBox, allez l'implémenter et revenez.

Configuration du serpent

Ouvrez BorderSnake.java. Vous constaterez que le main() méthode est exactement la même que BorderBox. Cependant, certaines variables de classe sont différentes. Au lieu d'un seul GRect, nous avons maintenant une liste de tableaux de GRects. Nous avons toujours une direction pour le serpent, cependant.

private ArrayList<GRect> snake;
private int snakeDirection = EAST; // direction de départ

Mettre en place makeSnake(). Pour vous faciliter la vie plus tard, ajoutez d’abord la tête du serpent (pour que snake.get(0) renvoie le GRect pour la tête). Rappelez-vous que le serpent commence à faire face à EST, la tête est donc la plus à droite. Notre serpent est SNAKE_LENGTH = 5 blocs de long.

Se déplacer vers l'est

Définir frontIsClear() pour toujours retourner vrai pour le moment, afin que nous puissions tester notre serpent se déplaçant dans une direction (est). Mettre en place moveSnake() déplacer le serpent d'une case à droite; N'oubliez pas que le serpent doit être déplacé SQUARE_LENGTH + SQUARE_GAP afin de laisser un espace entre les différentes parties du serpent!

Il y a un truc ici; vous n'avez pas besoin de déplacer chaque bloc du serpent. En fait, il vous suffit de déplacer un bloc! Pensez au bloc que vous souhaitez déplacer pour donner l’illusion de l’ensemble du mouvement du serpent.

Le bloc que vous avez choisi sera maintenant la tête du serpent. Vous devez donc non seulement mettre à jour son emplacement sur la grille, mais également mettre à jour sa position dans ArrayList. Retirez le bloc et ajoutez-le à la tête (dans la partie précédente, nous avons désigné la tête comme étant la 0ème position de la liste de tableaux).

snake.remove(block); // le GRect que vous avez déplacé
snake.add(0, block); // devient la nouvelle tête

Ensuite, implémenter frontIsClear() pour vérifier quand vous avez atteint le mur le plus à droite. Encore une fois, il y a un truc ici: pouvez-vous vérifier si tout le serpent peut se déplacer en ne vérifiant qu'un seul bloc? (Indice: c'est la tête)

Vous devriez maintenant être en mesure de voir votre serpent se déplacer pas à pas vers le mur est!

Se déplacer le long de la frontière

Vous êtes très proche de la ligne d'arrivée !! Mettre en place turnLeft(). Cela devrait être très simple. souvenez-vous de ce que nous avons fait dans BorderBox.

Maintenant, nous devons prolonger moveSnake() et frontIsClear() pour gérer plusieurs directions. Il y a un dernier tour. Supposons que nous ayons le serpent dans la configuration suivante, où snakeDirection = NORTH:

Pour déplacer le serpent (lisez: donnez l'illusion que tout le serpent a bougé), est-ce que nous nous inquiétons du fait qu'une partie du serpent est toujours en train de tourner? (Indice: non, pas du tout) À partir de cette information, comment déplacer le bloc que vous avez sélectionné précédemment dans la partie précédente pour se déplacer vers le nord, dans la direction à laquelle le serpent est confronté?


Félicitations, vous avez maintenant un serpent en mouvement! :)
