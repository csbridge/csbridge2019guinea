Serpent Frontière

Écrit par Lisa Yan

Référence: Référence graphique
ArrayLists

Ceci est un programme de bonus! C'est un peu difficile, mais terminer ce programme vous aidera certainement à écrire votre projet final.

Ecrivez un programme graphique qui déplace un serpent dans le sens antihoraire autour du bord de la toile.

BoîteFrontière

Cet exercice suppose que vous avez terminé BorderBox. La logique est similaire (il n’ya qu’une astuce supplémentaire), il est donc très utile de connaître la configuration générale du problème. Si vous n'avez pas encore terminé BorderBox, allez l'implémenter et revenez.

Trucs et astuces

  • Utilisation ArrayLists!
  • Lorsque vous déplacez le serpent, vous n'avez pas besoin de déplacer chaque bloc du serpent. En fait, il vous suffit de déplacer un bloc! Pensez au bloc que vous souhaitez déplacer pour donner l’illusion de l’ensemble du mouvement du serpent.

Invariants

Comme indiqué ci-dessus, dans ce programme, il vous suffit de déplacer un seul bloc pour donner l’illusion que tout votre serpent a été déplacé. Chaque fois que le serpent se déplace, différents blocs dans ArrayList représentent désormais la nouvelle tête et la nouvelle queue du serpent. Comment pouvez-vous accéder de manière fiable à la tête de votre serpent dans votre programme?

Une option que vous avez est de définir une logique invariant que la tête de votre serpent sera toujours la 0ème position de votre ArrayList. Un invariant est un terme informatique qui désigne quelque chose qui peut toujours être supposé être vrai. Bien qu'il n'y ait pas d'analogie exacte avec le codage, cette compréhension logique aidera le codeur à exécuter son programme. Le code ci-dessous aide à traduire l'invariant logique dans le code que vous devez faire:

snake.remove(block); // le GRect que vous avez déplacé
snake.add(0, block); // devient la nouvelle tête

Notez que définir cet invariant vous aidera également à déterminer quelle position de la liste de tableaux sera toujours la queue du serpent.
